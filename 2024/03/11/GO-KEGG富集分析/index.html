GO-KEGG富集分析

无标签
发布日期:   2024-03-11

你上传的图片似乎是两个不同来源的基因富集分析结果的截图，一个来自服务器，另一个使用R语言生成的。这些分析有助于识别与一组基因显著相关的生物学过程、细胞组分和分子功能。

通过比较这两组结果，我可以观察到以下区别：

  1. 基因本体术语：虽然在识别的生物学过程中有一些重叠（例如，细胞周期过程和染色体分离），但在列出的具体基因本体（GO）术语上可能有差异。服务器基础的分析列出了更详细的GO术语。

  2. 统计值：

    • 数量和百分比：服务器基础的分析提供了与每个术语相关的基因数量以及这个数量占分析总基因数的百分比。
    • P值和调整后的P值：两种分析都提供了p值和调整后的p值，但调整方法可能有所不同，这可能会影响术语的显著性水平。服务器分析使用了Bonferroni和Benjamini方法，而R分析报告了p.adjust和qvalue，这可能对应不同的统计校正方法。
    • 富集：服务器基础的分析包括倍数富集，这有助于了解与特定GO术语相关的基因出现的频率比随机预期的频率高多少。

  3. 数据呈现：

    • 服务器分析包含了“倍数富集”、“Bonferroni”、“Benjamini”和“FDR”列，这些是不同的统计量度，用来考虑多重比较的影响。
    • R分析似乎侧重于“p.adjust”和“qvalue”列来调整多重比较的p值。

  4. 基因列表：

    • 两种分析的每个术语的基因列表可能有差异。这可能是由于不同的基因集数据库、不同版本的基因本体或不同的纳入阈值所致。
    • 服务器分析包括了一个“列表总数”列，而R分析中这个列不明显。

  5. 背景集：

    • R分析提供了一个“BgRatio”，这可能代表在参考集中与每个术语相关的基因的背景比率，这对于理解富集的背景很重要。

  6. 格式：

    • 服务器分析用颜色编码进行了标记，以便更容易解释最显著的结果。
    • R分析呈现了更标准的表格格式，没有颜色编码。

总之，虽然两种分析基于同一组基因，但它们可能使用不同版本的GO数据库、不同的统计测试或显著性阈值，这可能导致GO术语及其相关统计数据有轻微的差异。

R能用哪两种方法吗?

是的，R可以使用Bonferroni和Benjamini-Hochberg方法来调整多重比较的p值。这些方法是校正假阳性发现率的常见手段，它们在许多R统计和生物信息学包中都有实现。例如，在R中使用p.adjust函数就可以应用这些方法：

其中p.values代表你的原始p值向量。method参数指定了要使用的调整方法。Bonferroni方法非常保守，适用于控制家族错误率（Family-wise Error Rate, FWER），而Benjamini-Hochberg方法则控制假发现率（False Discovery Rate, FDR），通常被认为更加强大。在基因富集分析等领域，Benjamini-Hochberg方法更为常用，因为它在降低假阳性的同时保留了更多的真阳性发现。

服务器和R语言提供的富集分析结果的列标题具有一些相似性，但也有差异。以下是每个标题所代表的含义：

服务器的结果列:

  • Category: 这可能是指GO的分类，如生物学过程（BP）、细胞组成（CC）或分子功能（MF）。
  • Term: GO术语或KEGG路径的描述。
  • Count: 在该术语下发现的基因数量。
  • %: 基因数量占分析中总基因数的百分比。
  • PValue: 术语富集的原始p值。
  • Genes: 与该术语相关联的基因列表。
  • List Total: 分析中的基因总数。
  • Pop Hits: 在参考基因集中与该术语相关的基因数量。
  • Pop Total: 参考基因集的总基因数量。
  • Fold Enrichment: 富集倍数，指的是观察到的与期望的比例。
  • Bonferroni: 使用Bonferroni方法校正后的p值。
  • Benjamini: 使用Benjamini-Hochberg方法校正后的p值。
  • FDR: 假发现率，是经过校正的p值，用于控制类型I错误。

R语言的结果列:

  • ONTOLOGY: 同上，指GO的分类，BP、CC或MF。
  • ID: GO术语或KEGG路径的ID。
  • Description: 术语或路径的描述。
  • GeneRatio: 在该术语下发现的基因数量与分析基因总数的比例。
  • BgRatio: 背景比率，可能是指该术语在背景基因集中的基因数量比例。
  • pvalue: 术语富集的原始p值。
  • p.adjust: 调整后的p值，可能默认是使用了某种校正方法，如Benjamini-Hochberg方法。
  • qvalue: 类似于FDR的一个校正后的p值，它提供了发现被错误地声明为显著的比例的估计。
  • geneID: 与该术语相关的基因ID列表。
  • Count: 同上，基因的数量。

虽然两者提供的信息类似，但具体的表格格式和某些统计数据的计算可能有所不同。这可能是因为不同的软件和包在处理富集分析时采用了不同的方法和默认设置。在实际应用中，研究者应根据实验设计和研究目标选择合适的方法和参数。

R语言代码

要在图表的右侧添加p-value的信息，它显示的adjust的信息

GO<-enrichGO( gene$ENTREZID,#GO富集分析
OrgDb = GO_database,
keyType = “ENTREZID”,#设定读取的gene ID类型
ont = “ALL”,#(ont为ALL因此包括 Biological Process,Cellular Component,Mollecular Function三部分）
pvalueCutoff = 0.05,#设定p值阈值
qvalueCutoff = 0.05,#设定q值阈值
readable = T
)

除了这些参数还有什么参数吗？

是的，enrichGO函数提供了多个参数来细化GO富集分析，除了你提到的这些参数外，还包括但不限于以下一些：

  • universe：用于指定背景基因集，即可能出现的所有基因。如果不指定，分析会默认使用OrgDb中的所有基因作为背景集。

  • minGSSize 和 maxGSSize：分别用于设置富集分析中考虑的基因集的最小和最大尺寸。这可以帮助过滤掉太小或太大的基因集，这些基因集可能因为尺寸原因而不具代表性或统计意义。

  • qvalueCutoff：用于设置q值（校正后的p值）的截断值，用于控制假发现率(FDR)。

  • pAdjustMethod：用于调整p值的方法。包括”holm”、”hochberg”、”hommel”、”bonferroni”、”BH”（也称为”fdr”）、”BY”等，这些方法可以控制多重检验的错误率。

  • evidence：允许用户根据证据代码过滤GO条目，例如通过特定实验验证的条目。

  • ont：指定进行富集分析的本体类别，包括”BP”（生物过程）、”CC”（细胞组分）和”MF”（分子功能）以及”ALL”。

  • readable：当设置为TRUE时，会尝试将基因ID转换为更易读的基因名称。

  • pool：如果设置为TRUE，则会汇总不同本体的相同条目，这可能有助于简化输出结果。

这些参数允许用户根据具体的研究需求调整分析的细节。选择合适的参数设置可以帮助用户获得更准确和具有解释性的结果。为了深入了解这些参数及其用法，建议参阅DOSE包的官方文档或使用?enrichGO命令查看更详细的帮助信息。

一般情况下，用什么，给我代码

library(clusterProfiler)
library(org.Hs.eg.db) # 以人类基因为例，如果是其他物种请替换相应的OrgDb包

假设你已经有了一个基因列表（Entrez ID格式）

gene_list <- c(“geneID1”, “geneID2”, “geneID3”, …) # 请用实际的基因ID替换这里的示例ID

进行GO富集分析

result <- enrichGO(gene = gene_list,
OrgDb = org.Hs.eg.db,
keyType = “ENTREZID”,
ont = “ALL”, # 包括生物过程(BP)、细胞组分(CC)和分子功能(MF)
pAdjustMethod = “BH”, # 控制假发现率，”BH”即Benjamini-Hochberg方法
pvalueCutoff = 0.05, # p值截断
qvalueCutoff = 0.05, # q值截断，控制假发现率
readable = TRUE) # 转换基因ID为可读的基因名

查看富集分析结果

head(result)

可视化富集分析结果

library(DOSE)
barplot(result, showCategory=20) # 显示前20个显著的GO项
